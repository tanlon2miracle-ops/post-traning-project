# 数据合成方案 & Prompt 模板

> 版本: v1
> 状态: 模板已设计，待实际数据验证

## 概述

**目标**: 从有限的人审数据（上万条）+ 规则库，扩充到 5-10 万条高质量 SFT 数据

**合成工具**: Distilabel（接内部 API：GLM / Qwen3）

**两条合成线**:
1. **违规样本扩充**（reject）：基于现有数据 + 规则，生成更多样的违规变体
2. **合规样本扩充**（pass）：生成容易被误判但实际合规的"边界样本"

## SFT 数据格式

```json
{
  "messages": [
    {
      "role": "system",
      "content": "你是一个内容安全审核专家，负责判定用户发布的内容是否违规。"
    },
    {
      "role": "user",
      "content": [
        {"type": "image", "image": "path/to/image.jpg"},
        {"type": "text", "text": "请审核以下内容：\n用户发布文本：「XXX」"}
      ]
    },
    {
      "role": "assistant",
      "content": "判定：违规\n标签：色情低俗\n理由：该文本包含明显的色情暗示内容。\n置信度：高"
    }
  ]
}
```

## 合成 Prompt 模板

### 模板 1：种子样本变体生成

从人审数据中挑选种子，生成表述不同但违规本质相同的变体。

```
你是内容安全数据标注专家。以下是一条真实的违规内容样本：

【原始文本】{seed_text}
【违规标签】{label}
【违规原因】{reason}

请生成 5 条与此样本违规本质相同，但表述方式、语气、场景不同的变体。要求：
1. 每条变体的违规程度应有梯度（从明显到擦边）
2. 模拟真实用户发布风格（口语化、有错别字、用谐音字、缩写等）
3. 包含对抗性写法（故意绕过关键词过滤的手法）
4. 输出格式：JSON 数组，每条包含 text, label, severity(high/medium/low), evasion_technique(如有)
```

**预估产出**: 每种子 5 条 × 200-500 种子/标签 = 5,000-12,500 条

### 模板 2：规则正反样本对生成

从规则库提取规则，生成"刚好触发"和"刚好不触发"的样本对。

```
你是内容安全规则专家。以下是一条审核规则：

【规则名称】{rule_name}
【规则描述】{rule_description}
【触发条件】{trigger_conditions}
【豁免条件】{exemption_conditions}

请生成：
A. 3 条明确触发该规则的违规内容（不同场景和表述）
B. 3 条看起来相似但实际不触发的合规内容（边界样本）
C. 2 条需要人工判断的模糊样本

每条包含：text, is_violation(true/false), label, reasoning
```

**预估产出**: 每规则 8 条，视规则数量

### 模板 3：场景化合成

按平台场景生成特定风格的样本。

```
你是一个{platform_scene}的内容模拟器。请生成以下类型的内容样本：

场景：{scene_description}
目标标签：{target_label}
数量：10 条

要求：
1. 符合{platform_scene}的语言风格和长度特征
2. 包含 7 条违规 + 3 条合规（边界样本）
3. 违规样本中要有 2-3 条使用了常见的逃避检测手法
4. 输出 JSON，每条包含 text, scene, label, is_violation
```

**场景池**:
- 商品评论（短文本、口语化）
- 商品描述（营销体、夸张用语）
- 直播弹幕（极短、缩写、emoji 多）
- 用户私信（对话体）
- 投诉/举报内容
- 商家回复

**预估产出**: 6 场景 × 5 标签 × 100 条 = 3,000 条

### 模板 4：对抗性样本生成

专门生成绕过传统规则引擎的对抗样本。

```
你是一个红队测试专家。以下是当前审核系统的规则：

【被检测的关键词/模式】{detected_patterns}
【标签】{label}

请生成 10 条违规内容，使用以下逃避技术（每种至少 2 条）：
- 谐音替换（如：约炮→约泡→约跑）
- 拼音/首字母缩写
- emoji 替代关键词
- 拆字/间隔插入无意义字符
- 隐晦暗示（不含关键词但意图明确）

输出 JSON：text, label, evasion_technique, difficulty(easy/medium/hard)
```

**预估产出**: 每标签 200 条 = 1,000 条

### 模板 5：CoT 推理链生成

为助手回复生成详细的审核推理过程。

```
请作为内容安全审核员，对以下内容进行审核并给出详细推理：

【待审内容】{content}
【图片描述】{image_description}（如有图片）

请按以下格式输出：
1. 初步印象：（第一感觉是否违规）
2. 关键词/特征分析：（识别到的风险信号）
3. 上下文判断：（结合图文综合分析）
4. 规则匹配：（匹配到哪条规则）
5. 最终判定：违规/合规
6. 标签：{label_options}
7. 置信度：高/中/低
8. 理由：（一句话总结）
```

**用途**: 后处理阶段，为所有合成样本生成高质量助手回复

## 合成流水线

```
Step 1: 种子挑选
├── 从人审数据中每个标签抽 200-500 条高质量种子
└── 人工确认种子质量

Step 2: 批量合成
├── 模板 1（变体）：5,000 - 12,500 条
├── 模板 2（规则正反对）：视规则数量
├── 模板 3（场景化）：3,000 条
├── 模板 4（对抗性）：1,000 条
└── 模板 5（CoT）：后处理，为所有样本生成推理

Step 3: 质量过滤
├── 交叉验证（GLM 合成 → Qwen3 验证，反之亦然）
├── 去重（语义去重，Data-Juicer）
├── 人工抽检 5-10%
└── 过滤：标签一致率 < 85% 的丢弃

Step 4: 混合
├── 合成风控数据 60%
├── 开源通用数据 40%
└── 打乱 + 格式化 → 最终 SFT 训练集
```

## 质量控制

| 指标 | 标准 | 方法 |
|------|------|------|
| 标签一致率 | ≥ 85% | 交叉验证（GLM ↔ Qwen3） |
| 多样性 | 无大量重复 | 语义嵌入聚类检查 |
| 难度分布 | easy 40% / medium 40% / hard 20% | 人工+自动评估 |
| 人工抽检通过率 | ≥ 90% | 随机抽检 5-10% |
