# 标签体系设计

> 版本: v1 (MVP)
> 状态: 已确定，待数据验证

## 设计原则

1. 从少量标签开始验证效果，后期按需拆分
2. 现有人审数据标签粒度细但区分度不够，需先合并
3. 一期为**单标签分类**（非多标签）

## 一期标签体系（5 类）

| 一级标签 | ID | 覆盖的原始标签 | 说明 |
|---------|-----|--------------|------|
| 色情低俗 | `pornography` | 色情、软色情、低俗擦边 | 性暗示、裸露、低俗内容 |
| 欺诈违规 | `fraud` | 诈骗话术、刷单、薅羊毛 | 经济欺诈类 |
| 违禁内容 | `prohibited` | 政治敏感、暴力恐怖 | 法规明确禁止的内容 |
| 平台违规 | `platform_violation` | 导流、辱骂、平台负面、商品负面 | 平台规则类违规 |
| 正常 | `pass` | 合规内容 | 无违规 |

## 原始标签 → 一级标签映射

```
色情        → 色情低俗
软色情      → 色情低俗
低俗擦边    → 色情低俗
诈骗话术    → 欺诈违规
刷单        → 欺诈违规
薅羊毛      → 欺诈违规
政治敏感    → 违禁内容
暴力恐怖    → 违禁内容
导流        → 平台违规
辱骂        → 平台违规
平台负面    → 平台违规
商品负面    → 平台违规
pass/合规   → 正常
```

## 待讨论

- [ ] 色情和政治的细分是否够？（原始标签也比较粗）
- [ ] 是否需要二级标签？（一期先不做，看效果）
- [ ] "商品负面" 归入 "平台违规" 是否合理？部分场景可能是合理差评
- [ ] 置信度是否需要作为输出？（当前人审数据没有置信度）

## 后期拆分方向（Phase 2+）

如果一期效果不错，可拆分为更细粒度：

```
色情低俗
├── 色情（直接暴露）
├── 软色情（暗示/擦边）
└── 低俗（非色情但低质）

欺诈违规
├── 诈骗话术
├── 刷单
└── 薅羊毛

违禁内容
├── 政治敏感
├── 暴力恐怖
└── 违法广告

平台违规
├── 导流/引流
├── 辱骂/人身攻击
├── 平台负面舆论
└── 商品负面（需区分合理差评 vs 恶意抹黑）
```
